Settlement Attestation Layer for RWA — on-chain compliance bridge for stablecoin real-world-asset settlements.

Problem

RWA settlements grew 8×. Compliance did not.

  • Stablecoin is the default rail for cross-border RWA in 2026
  • Tokenization is solved — Centrifuge, RealT, Ondo, Maple
  • Banks have no standard way to signal bank-grade on-chain
  • Buyers wire USDC to unverified payees · regulators lack audit trails
AttestRWA2 / 7
Solution

One attestation schema. Any RWA. Any bank.

On-chain
  • SettlementApproval EAS schema (10 fields)
  • SettlementEscrow.sol — release only on valid attestation
Off-chain
  • FastAPI attester — payee, capital class, evidence
  • Compliance YAML DSL — bank policy as code

We are the composable layer tokenization platforms plug into — not another RWA token.

AttestRWA3 / 7
How it works

Attest → enforce → settle

Buyer wallet → deposits Mock USDC → SettlementEscrow
Attester service → checks payee feed + capital + RAG evidence
Attester signs → EAS SettlementApproval on Base
Escrow reads attestation → release to verified payee
OR payee mismatch → reject → buyer refund · full audit trail on-chain
